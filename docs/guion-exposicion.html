Parte 1 — Introducción y arquitectura   (0:00 – 3:00)

P1 · Max García Campos
"Buenos días, somos Max García y Erick Nogales. Vamos a presentar Lo de Juan, una aplicación móvil para la gestión de pedidos, punto de venta y delivery de una pollería en Lima." 📂 [README.md] "El problema que resolvemos: la toma de pedidos en papel genera errores, demoras y sin visibilidad del estado en tiempo real. Nuestra app centraliza todo en un sistema con tres roles: cliente, empleado y administrador." 📂 [app/_layout.tsx] "La app está construida con React Native y Expo SDK 54, usando Firebase como backend. En el layout raíz se ven los tres providers que envuelven toda la aplicación:"
(AuthProvider → sesión y roles)   (CarritoProvider → carrito en memoria)   (DeliveryProvider → datos de delivery entre pantallas) 📂 [src/contexts/AuthContext.tsx] "Firebase Auth se conecta así: al iniciar sesión, el contexto escucha el estado con onAuthStateChanged, recupera el rol desde Firestore y lo expone como booleanos isAdmin, isEmployee, isCustomer. La navegación se adapta sola según quien entra." 📂 [app/(tabs)/_layout.tsx] "Acá se ve en acción: la barra de tabs se reconfigura dinámicamente. El cliente nunca ve el POS, el empleado no ve el home de compras. Cada tab usa href: null cuando el rol no corresponde. Esa fue una decisión de usabilidad clave: reducir la carga cognitiva mostrando solo lo relevante para cada persona."

Parte 2 — Flujo del cliente   (3:00 – 6:00)

P2 · Erick Nogales
"Ahora vemos el flujo completo del cliente." 📂 [app/(tabs)/index.tsx] "Esta es la pantalla principal. Los productos se cargan desde Firestore con getProductosActivos(), agrupados por categoría. El cliente agrega al carrito directo desde acá."
(handleAddItem → llama al contexto del carrito)   (getCartQty → sincroniza la cantidad mostrada en tiempo real) 📂 [app/carrito.tsx] "En el carrito el cliente elige el método de pago: Yape o tarjeta. Al tocar 'Pagar' se abre el modal de delivery donde elige entre recoger en tienda, delivery a domicilio o consumo en mesa." 📂 [src/components/DeliveryModal.tsx] "El modal tiene tres pasos. En el paso de delivery se muestra un mapa interactivo con Leaflet dentro de un WebView. El cliente puede buscar su dirección o mover el pin, y también puede guardar la dirección para usarla la próxima vez." 📂 [app/pago-yape.tsx] "En el pago con Yape se muestra el QR real de la pollería con instrucciones paso a paso."
(handleYaPague → llama a createOrder en Firestore)   (descontarStock → usa increment() atómico para restar el inventario)   (saveUserAddress → guarda la dirección del cliente automáticamente)

Parte 3 — Flujo del empleado y administrador   (6:00 – 9:00)

P1 · Max García Campos
"Pasamos al lado operativo. Primero el empleado." 📂 [app/(tabs)/pedidosAdmin.tsx] → función ComandaCard "Los empleados ven las comandas en formato de ticket de cocina. Cada comanda tiene un temporizador en tiempo real que cambia de color: verde si lleva menos de 5 minutos, amarillo hasta 15, rojo después. Eso ayuda a priorizar."
(useElapsed → calcula tiempo desde createdAt con setInterval cada segundo)   (getUrgencyColor → retorna el color según minutos transcurridos)

"Con un toque en el botón cambian el estado del pedido: pendiente → preparando → listo. Simple y rápido para cocina." 📂 [app/(tabs)/pos.tsx] "Para el administrador, el POS permite crear pedidos desde caja. Hay filtro por categoría, control de stock en tiempo real, y al cobrar se elige entre mostrador o mesa con número." 📂 [app/(tabs)/admin/index.tsx] "El panel admin muestra métricas en tiempo real: total de productos, categorías, usuarios, pedidos pendientes y ventas del día. Desde acá accede a todos los módulos de gestión."
(load → Promise.all con getProductos, getCategorias, getDocs, getOrders simultáneos) 📂 [app/(tabs)/admin/productos.tsx] "El CRUD de productos es completo: crear, editar con imagen por URL, activar o desactivar, y controlar cantidad mínima. Lo mismo existe para categorías, usuarios y el módulo de capacitación."

Parte 4 — Pruebas de usabilidad y cierre   (9:00 – 12:00)

P2 · Erick Nogales
"Pasamos a las pruebas de usabilidad. Realizamos sesiones moderadas presenciales con dos participantes, evaluando seis tareas: login, agregar al carrito, ajustar cantidades, pagar con Yape, ver el estado del pedido y acceder a capacitación. Encontramos cuatro problemas principales."
Problema 1: Los usuarios tocaban la imagen del plato en vez del botón "Agregar". 📂 [app/(tabs)/index.tsx] → función ProductCard "La solución fue envolver toda la tarjeta del producto en un TouchableOpacity. Antes solo respondía el botón '+'."
Problema 2: No encontraban el acceso a Capacitación porque era un tab condicional. 📂 [app/(tabs)/_layout.tsx] "La solución fue asegurar que el tab de Capacitación siempre esté visible para empleados y admins, sin condición de ocultarlo."
Problema 3: Confusión con el QR de Yape, preguntaban si debían abrir la app real. 📂 [app/pago-yape.tsx] "Se agregaron instrucciones numeradas paso a paso y el botón 'Ya pagué' se hizo más prominente."
Problema 4: No era evidente que podían cambiar cantidades en el carrito. 📂 [app/carrito.tsx] "Se agregaron los controles +/− visibles directamente en cada ítem de la lista."

"Para cerrar: la app cumple con todos los criterios de la rúbrica. Está construida con React Native y Expo, tiene navegación completa por tabs con control de roles, Firebase Auth con tres roles, CRUD completo en Firestore, y mejoras documentadas basadas en pruebas reales de usuario. El repositorio está disponible en GitHub con el README de instalación. Gracias."
